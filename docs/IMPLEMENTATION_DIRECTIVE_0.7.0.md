## 7. 画像の一枚単位承認制

すべての新規画像は、一枚ごとにプロデューサーの明示承認を得る。

対象：

- キャラクター基準アート
- ポートレート
- 会話立ち絵
- 表情差分
- 戦闘スプライトシート
- 敵スプライトシート
- ボス画像
- 背景
- ステージオブジェクト
- 重要イベント一枚絵
- その他ゲーム画面へ表示する視覚アセット

承認手順：

1. 同時に複数画像を承認依頼しない。
2. 一度に一枚だけ提示する。
3. Asset ID、用途、予定パス、revision、短いデザイン意図、確認事項を添える。
4. `この画像で進めてよいですか`と確認する。
5. 回答は`承認`、`修正：内容`、`却下`として扱う。
6. 明示承認前に正式採用、派生画像制作、ゲーム統合、次画像提示へ進まない。
7. 修正版は同Asset IDの新revisionとして再提示する。
8. 却下画像を正式・仮アセットや背景素材へ流用しない。
9. 承認履歴をIssue #37またはPR #38へ記録する。
10. asset manifestへ承認者、承認日時、revision、commit、最終パスを記録する。

進行速度：

- 画像承認待ちの間も、save、AI、UI、戦闘ロジック、データ、テストなど画像非依存作業は並行継続する。
- 承認待ち画像に依存する統合工程だけを停止する。
- 圧縮、形式変換、透明padding、pixel-identicalなリサイズは新規デザイン承認と扱わない。
- 最終ゲーム画面での表示QAは必須。
- 複数画像を一覧でまとめて承認したものとして扱わない。

承認依頼テンプレート：

```md
## 画像承認依頼 1件

- Asset ID：
- 用途：
- 予定パス：
- Revision：
- デザイン意図：1〜3行
- 確認：人物同一性／世界観／役割識別／スマホ視認性／破綻の有無

[画像1枚]

この画像で進めてよいですか。
回答：`承認`／`修正：...`／`却下`
```

## 8. 戦闘スプライト

- 制作基準方向は右向き
- 待機、移動A、移動B、攻撃予備、攻撃、被弾を最低限持つ
- 死亡を被弾と共用して品質が落ちる場合は別状態
- 6セル入力を派生atlasへ変換する方式、または7状態×左右2方向atlasのどちらでもよい
- 全フレームで同じ顔、髪、服、体格、武器、配色を維持
- 足元、頭の高さ、縮尺を統一
- 透明gutterを設け、セル間補間漏れを防止
- 武器、手足、装備を切らない
- 非対称装備が反転で破綻する場合は左右専用フレーム
- manifest、content rect、anchor、hitboxは実画像から計測

## 10. 7枠編成と複数召喚

- 最大7枠を開始時から利用可能
- 空き枠あり、最低1名で出撃可能
- 編成カードは人物ごとに1種類
- 7枠へ同じ人物カードを重複登録しない
- 選択済みカードからコストと再出撃待ちに従って同じ人物を複数体同時召喚可能
- 戦闘不能後も再出撃可能
- 永久離脱、死亡持越し、通常モードの人物ロックなし
- 3プリセット：`部隊1`、`部隊2`、`部隊3`
- 所有していない人物を含む旧プリセットは該当枠を空欄へ移行
- スマートフォンで7カードを豆粒化しない

## 12. 前進・後退・手動戦術機能の完全撤去

前進・後退および同目的の手動移動・戦術モードは、追加しないのではなく撤去する。

撤去対象：

- 前進・後退ボタン
- 防御・標準・突撃など同目的のボタン
- 関連キーボード・タッチ入力
- 関連チュートリアル、説明文、ARIA、ヘルプ
- 関連UI状態、コマンド消費、クールタイム、保存項目
- 関連専用テスト

ユニット配置コスト、再出撃待ち、支援ゲージなど別用途の資源は誤って削除しない。共用定数・状態の用途を調査し、手動戦術専用部分だけを撤去する。

撤去後に空白、使えないボタン、透明なタップ領域、死んだショートカットを残さない。

## 13. 味方AI

手動戦術機能の代替として自律AIを改善する。

- 攻撃可能な敵がいる場合、敵を無視して前進しない
- 目の前、接触中、通行阻害、CRAWLER攻撃中の敵を優先
- 敵とhitboxが重なる、または通過を阻まれている状態で前進を継続しない
- TAKUYA撃破後も残存敵、後続敵、CRAWLER攻撃中の敵を有効標的として扱う
- Y座標ずれ、レーン移動、target消失後に次tickで再取得
- 遠距離、近接、ナオの実攻撃条件と索敵条件を一致
- 敵を素通りして攻撃せず死亡し、敵が増え続ける状態を再発させない

必須回帰テスト：

- Stage 1〜3
- 全既存ユニット種
- 3レーン
- 敵が正面、背後、CRAWLER付近、レーン移動中
- Stage 3 TAKUYA撃破直後
- 残存敵と後続敵が同時にいる状態
- 味方前進中に敵が射程へ入る状態
- 近接、遠距離、ナオの別

実ブラウザQAで敵素通り再現0件を確認する。
