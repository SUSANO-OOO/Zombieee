# ASHFALL OUTPOST — 企画・仕様引き継ぎ

更新日：2026-07-13

## 1. 目的

この文書は、ASHFALL OUTPOSTの作品意図、長期構想、承認済みの製品・ゲーム仕様を共有する正式な所有元である。

- 運用・承認ルール：`AGENTS.md`
- 実装順・依存関係：`docs/PRODUCT_ROADMAP.md`
- 現在状態：`docs/PROJECT_STATE.md`
- 個別タスク：GitHub Issue

## 2. 作品概要

ASHFALL OUTPOSTは、生存者が暮らす大型移動拠点CRAWLERを守りながら部隊を指揮し、感染体の群れを押し返して敵拠点を破壊するリアルタイム戦略・防衛ゲームである。

主対象はスマートフォン横画面、PC横画面も正式対応。現行の正式戦闘構造は3レーン。

## 3. 完成時の基本ループ

**戦闘  
→ 報酬・恒久通貨獲得  
→ 新ユニット購入・解放  
→ 強化・装備・アビリティ  
→ 部隊編成・ロードアウト選択  
→ 盤面上の次ステージへ挑戦  
→ 敵・ユニット・物語・図鑑情報を解放**

戦闘内資源と戦闘外の恒久通貨を分離する。

## 4. キャンペーン

- 複数の盤面・地域・チャプター
- 各盤面に複数ミッション
- 進行に応じて難易度上昇
- クリア済みステージは自由に再プレイ
- 初回報酬は再クリア報酬より大きい
- 一部盤面クリアで特定ユニットを確定解放
- 正確な敵構成は隠し、脅威カテゴリだけ表示
- 通常ステージとボスステージを分離
- 通常は敵拠点破壊を基本にしつつ、防衛・護衛・救助などへ拡張

現在のTAKUYA戦は最初の正式ボスステージとする。

## 5. ストーリー

- 会話、ナレーション、ゲーム内グラフィックによるイベントシーン
- 二択などの選択肢
- 多くの選択は盤面・チャプター内で完結
- 少数の重要選択だけ後半の敵勢力、ボス、報酬、解放ユニット、エンディングへ影響
- ストーリー専用キャラクターと戦闘ユニットを分離可能
- ストーリー上で離脱・敵対しても、所有済み戦闘ユニットとしては使用可能

## 6. ユニットと編成

- 長期的な到達目標は60〜70体
- 1.0.0で全数投入せず、更新・DLCで増やす
- 兵種・クラスを基本構造にし、同クラスから個性の異なる複数ユニットを作る
- 現行のSCOUT、RANGER、BREAKER、BRAWLER、GUNNER、MEDICは暫定表記
- 将来は7〜8種類を編成
- 同じユニットカードを複数枠へ登録しない
- 同クラスの別ユニットは複数編成可能
- 偏った編成も許可する
- 同じカードから戦闘中に何度でも召喚可能
- 同時出撃数にプレイヤー向け固定上限を設けない
- 強いユニットほど高コスト・長クールタイム
- 弱いユニットほど低コスト・短クールタイム
- 装備・強化・アビリティでコスト・クールタイムを軽減可能
- ゼロコスト化やクールタイム消失を防ぐ

## 7. 資源とロードアウト

### 指揮力
- ユニット召喚に使用
- 時間経過で自動回復
- 将来、敵が確率で指揮力アイテムを落とす
- タップで獲得し、一定時間後に消滅
- 装備・能力でドロップ率を補正可能

### スクラップ
- 戦場物資に使用
- 戦闘ごとにリセット
- 恒久通貨と分離

### 支援ゲージ
- 航空支援などの緊急支援に使用

### 戦場物資
- 戦術投下ポッド、爆薬ドラム、救急物資など
- 持ち込める種類は基本1種類
- スクラップがある限り繰り返し使用
- 将来は段階強化可能

### 緊急支援
- 一度解放した支援は消耗品にしない
- 将来は複数から1種類を選択
- 0.5.0では航空支援を固定

### 装備
- 個人装備はユニット本人へ保存
- 装備品には所持数がある
- 同じ装備を複数所有すれば所持数分だけ複数ユニットへ装備可能
- 個人効果と部隊全体効果を分ける
- 部隊全体装備は初期2枠を第一候補

### プリセット
- 将来、6個前後を候補に保存
- 編成ユニット、戦場物資、緊急支援、部隊全体装備、作戦方針を保存
- 個人装備は保存しない

## 8. CRAWLER

- 生存者が暮らしながら移動する大型拠点
- 本編では一台の主役CRAWLER
- 現在のバス型を最終デザインにしない
- 最初の正式機体は多数の大型タイヤを持つ装甲移動拠点
- 通常ステージでは左側中央付近を基本位置
- 画面外への見切れで巨大さと戦場面積を両立
- 固有攻撃は全レーン一斉掃射
- クールタイム制で、開始時は一部進行済み
- ボスにも有効だが軽減
- 将来は装甲・火力・指揮支援の成長とモジュール
- 別機体はチャレンジモードやDLCで使用

## 9. セーブ

- 初期は端末内ローカルセーブ
- バージョンと移行処理を持つ
- 正式版までにクラウドセーブ
- スマートフォンとPCで同期
- 端末変更、削除、競合、バックアップ、復元を考慮

## 10. 図鑑

### ユニット図鑑
外見、クラス、能力、人物紹介、入手条件など。

### 敵・モンスター図鑑
- 未遭遇：シルエット
- 初遭遇：基本情報
- 複数回遭遇・撃破：能力、弱点、背景
- ボス：複数回撃破で段階解放

## 11. チャレンジモード

- 本編盤面とは独立した連続ミッション
- 開始前の配置時だけ論理マスを表示
- 戦闘開始後は非表示
- 各ユニット1体
- 再召喚なし
- 撃破ユニットは次の1ミッションのみ使用不能
- その次から復帰
- 負傷状態はチャレンジモード内だけ
- 控え、連戦報酬、撤退判断
- 別CRAWLER使用の候補

## 12. レーンとマス

- 1.0.0までは3レーンを正式構造
- 通常モードで常時マス目を表示しない
- 4〜5レーン化は比較プロトタイプ後に判断
- 844×390の視認性・操作・性能を優先

## 13. DLC

将来、盤面、ステージ、物語、敵、ボス、ユニット、CRAWLER、戦場物資、緊急支援、図鑑、チャレンジ構成を追加できるデータ構造を持たせる。販売方式・価格・配信方法は未決定。
