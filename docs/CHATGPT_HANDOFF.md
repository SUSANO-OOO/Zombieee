## 4. 現行0.4.0の固定仕様

- スマホ横画面対応。縦画面では回転案内を表示する。
- 960×540のゲーム世界を基準とし、Canvasを表示領域とDPRに合わせる。
- 戦場は上・中・下の3レーン。ユニットはY座標を使ってレーン間を移動できる。
- 味方はCRAWLERの出撃ベイから順次出撃する。
- 出撃待ちは最大3枠。
- 敵の最終目標はCRAWLER。
- 敵はCRAWLERへ前進し、進路を物理的に塞ぐ味方だけを迎撃する。
- 防衛・均衡・攻勢の3作戦を切り替えられる。
- TAKUYAはWAVE 8で登場するボス。
- TAKUYA撃破前は鉄柵を攻撃できない。
- TAKUYA撃破後、3レーンすべてを覆う鉄柵が攻撃可能になる。
- 鉄柵は全レーン共通のHPを持つ。
- 鉄柵HPが0になると勝利。
- CRAWLER HPが0になると敗北。両方が同時に0の場合も敗北を優先する。
- 6種類の味方ユニットと4種類の支援行動がある。
- 5秒間の準備時間がある。
- 一時停止、BGM、SFX、キーボード操作、タッチ操作に対応している。

## 6. ローカル限定の終盤QAモード

終盤回帰テスト用のQAモードがローカルコードに追加されている。

有効条件:

- ホストが `localhost` または `127.0.0.1`
- URLクエリが `?qa=endgame`
- 2条件を同時に満たす場合のみ有効

QAモードでは、TAKUYAが低HPで生存する終盤状態から開始し、通常の攻撃・死亡処理・鉄柵ダメージ・勝敗判定を通して終盤を検証できる。

QAモードは以下を直接設定して勝利を偽装していない。

- `barricadeVulnerable = true`
- 鉄柵HP = 0
- 勝利状態

公開ドメインでは、`?qa=endgame`を付けてもQAモードは有効にならないことを実ブラウザで確認済み。

## 9. ChatGPTとCodexの標準ワークフロー

当面はGitHubやWorkを追加せず、次の最小構成で運用する。

1. ユーザーが方向性や欲しい体験をChatGPTへ伝える。
2. ChatGPTは目的、プレイヤー体験、採用仕様、非対象、完了条件を整理する。

4. ユーザーが採用案を決める。
5. ChatGPTはCodexへ渡せる実装指示書を作る。
6. Codexは最初に現在のコードとGit状態を確認する。
7. Codexは必要最小限の変更を実装し、自動テストと実ブラウザ確認を行う。
8. ユーザーがCodexの完了報告をChatGPTへ共有する。
9. ChatGPTは、実装の成否ではなく「狙ったゲーム体験になったか」を評価する。
10. ユーザーの許可後にCodexがコミット・公開する。

ユーザーがChatGPTへ戻すものは、原則としてCodexの最終報告だけでよい。コード全文や会話全文は貼り直さない。画面の見た目を判断する作業では、必要に応じてスクリーンショットも添付する。バージョン公開や大きな設計変更の節目で、この引き継ぎ書を更新して再添付する。
