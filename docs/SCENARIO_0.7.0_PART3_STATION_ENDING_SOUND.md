# 3. サウンド実装正本

## 3.1 必須カテゴリ

- BGM
- 環境音
- 物語に関係するSE
- UI・操作SE
- 戦闘SE
- 短いキャラクターボイス
- 無音区間
- ボイスダッキング

## 3.2 導入

| 区間 | BGM | 環境音・SE | 演出 |
|---|---|---|---|
| 研究棟 | なし | 空調、蛍光灯、ケージ、隔離扉 | 電源断後に無音 |
| 救急搬入口 | なし | 救急車、首輪、犬の唸り | 咬傷を音で先に伝える |
| 西新の平常 | 街へ溶ける軽い曲 | 店、瓶、雨、地下鉄振動 | 徐々に不協和音 |
| 発生 | BGM停止 | クラクション、悲鳴、ガラス、火災報知器 | 音の密度を上げた後、停電で切る |
| タイトル | 短い専用曲 | 壊れた店内放送の残響 | 2秒無音後に開始 |

## 3.3 会話

- 通常会話開始時、BGMを3〜5dB下げる。
- 重要台詞では環境音も1〜3dB下げるが、完全には消さない。
- 台詞終了後250〜500msで復帰。
- 一行ごとに機械的な効果音を鳴らさない。
- 台詞にボイスがない場合でも、人物の動作音、衣擦れ、食器、工具などを適宜使う。
- 感情をBGMだけで説明しない。箸が落ちる、電話が切れる、息が止まるなどの具体音を優先する。

## 3.4 戦闘

- 重要警告、撤退、救助対象危機は攻撃SEより前へ出す。
- 多数の銃声、チェーンソー、フライパンが同時再生されてもクリップしない。
- 同一SEの連打はランダムピッチだけで誤魔化さず、同時発音数と優先順位を制御する。
- ボス段階変化は曲の差し替えではなく、層の追加・停止で自然につなぐ。
- スマートフォン本体スピーカー、イヤホン、PCで別々に聴感確認する。
- AudioContextの初回解除、バックグラウンド復帰、画面ロック復帰、BGM二重再生を確認する。

## 3.5 受入条件

- すべての指定場面でBGM、環境音、SEが意図どおり開始・停止する。
- 台詞が戦闘音やBGMへ埋もれない。
- 無音場面で残留ループがない。
- スキップ後に前の場面のBGMや環境音が残らない。
- 844×390、844×340の実機相当と物理スマートフォンで聴感確認する。

# 4. 五人格レビュー結果

## 日本語会話編集者
- 主語を必要以上に補わず、質問に即答しない反応、言い直し、沈黙を追加。
- 口癖だけで人物を分ける旧稿を不採用。
- 音読で長い説明台詞を分割またはト書きへ移した。

## ゲーム・ナラティブデザイナー
- 各会話の直後に、編成、救助、防衛、破壊、護衛、封鎖の行動が始まるよう接続。
- 加入人物は戦闘中に役割を見せ、勝利後の会話で関係を作る。
- 初回導入をタイトル画面の文章ではなくプレイ開始後の映像へ移した。

## ホラー演出家
- 人声模倣を中心設定から削除。
- 病院、感染動物、救急搬入口、西新の群衆という因果を映像と音で示す。
- 恐怖を怪物の説明ではなく、平常音が崩れる過程で作る。

## 群像劇脚本家
- パイセンの罪悪感、クマバーソンとTAKUYA、ガンテツとモンキーの関係を長期線に追加。
- 全員が即座に賛成する会話を削除し、撤退、救助、証拠保全で意見を衝突させた。
- 人物が場面の前後で同じ状態に戻らないよう修正。

## サウンドディレクター
- 各場面へBGM、環境音、SE、無音、ダッキングを明記。
- サウンドを後付け素材ではなく、流出、恐怖、救助、感情転換の証拠として設計。
- スマートフォンの音声復帰とミックスを受入条件へ追加。

五人格とも、旧v2は不採用、新v3を実装候補として承認。
