## 1. 実装契約

- プレイヤーは無言の指揮官。会話選択肢、性格分岐、固定台詞を追加しない。
- 指揮官の存在は、報告先、視線、地図マーカー、出撃・帰還操作で示す。
- Stage 4〜6の目的、勝利条件を脚本都合で変更しない。
- 戦闘中barkは、編成人物の場合は実際に出撃中の場合だけ表示する。
- Stage 4のガンテツ、Stage 5〜6のモンキーは、加入前の物語上の固定NPC支援として扱ってよい。プレイヤーの7枠カード、所有状態、重複ユニットには数えない。

- `医療支援`は正式名決定後に一括置換する役割ラベル。`センセイ`は使用しない。
- `レイダー`を正式表示名とし、`ロッカ`は使用しない。
- キャラクターボイスは使用しない。台詞は画面表示。

# 2. サウンド実装正本

## 2.1 基本

- キャラクターボイスは使用しない。
- 会話表示中は必要に応じてBGMを3〜5dB下げる。
- 重要台詞では環境音を1〜3dB下げるが、完全に消さない。
- 台詞表示終了後250〜500msで復帰する。
- これはテキスト演出のミックスであり、ボイスダッキングではない。
- 無音はプロローグのタイトル前、Stage 3最終局面、Stage 6封鎖完了後に限定する。
- スキップ後にBGM、警報、環境音ループを残さない。
- 戦闘中の警告bark表示と警告SEを攻撃演出へ埋もれさせない。

## 2.2 スマートフォン

- 初回ユーザー操作でAudioContextを解除する。
- 画面ロック、タブ復帰、回転後にBGMと環境音を二重再生しない。
- 本体スピーカーでBGM、環境音、SEを聞き分けられる。
- イヤホンでは低音と警報を過剰にしない。
- 音声有効化ボタンは押下状態と成功・失敗を視覚表示する。

## 2.3 完成条件

- 仮音源や単純なビープ音だけで完成扱いにしない。
- プロローグからStage 6まで、BGM、環境音、SE、無音が場面転換に追従する。
- スキップ、再視聴、敗北、再戦、ステージ帰還で音が正しく初期化される。
- キャラクターボイス、音声収録、ボイスダッキングを完成条件にしない。
