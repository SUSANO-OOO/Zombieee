# 西新世紀末物語 — 0.7.0物語・会話設計リサーチ

更新日：2026-07-17

## 1. 調査の範囲と誠実性

本調査は、119作品を比較対象として、公開されている公式紹介、開発者・脚本家インタビュー、プレイ映像・イベント映像、シナリオ解説、既知の作品構造を横断して分析したものである。
119作品を今回の作業時間内に新規で全編プレイ・全巻通読した、という虚偽の主張はしない。
台詞の転載は禁止転載や模倣は行わず、構造・演出・会話設計の原則だけを抽出する。

## 2. 参照した制作論・実務知見

- アニメ脚本は、監督やシリーズ構成の意図を踏まえたプロット、自然な会話、継続的なリテイクを重視する。
- ゲームの会話は文章だけでは完結せず、直前のプレイ、画面、表情、BGM、SE、次の操作との役割分担が必要。
- キャラクターらしい台詞は、甘さや派手さより、人物の価値観・関係・過去と一致していることが先。
- 台本は音読し、話者名を隠しても誰の台詞か判別できるか、息継ぎや反応が人間の会話として成立するかを確認する。
- サウンドはBGM、環境音、SE、ボイスを場面ごとに設計し、ボイス時のダッキング、無音、音源位置、復帰を含めて実装する。

## 4. 119作品から抽出した必須原則

### 4.1 導入

1. 最初に世界設定の説明を読ませず、平常が壊れる具体的な一事件を見せる。
2. 主人公が何を失い、誰を助けたいかを最初の10分以内に示す。
3. ウイルス名、研究計画、組織名は映像・記録・行動の中で段階的に出す。
4. タイトル前の導入シーンで「このゲームを続ける理由」を作る。
5. 王道の災害発生を恐れず、地域、人物、音、選択で固有性を作る。

### 4.2 世界観

1. 設定は人物が知っている範囲だけ語る。
2. 架空の研究機関を使用し、実在の大学病院が違法研究をしたように描かない。
3. 西新、早良区、商店街、飲み屋街、地下鉄を背景名ではなく、移動・避難・物資・人間関係へ結びつける。
4. Tウイルスは万能設定にせず、感染経路、発症、死亡、再活性化、変異条件を一貫させる。
5. TAKUYAとTウイルスの関係は、物語の途中で発見する謎として扱う。

### 4.3 会話

1. 一つの場面で各人物に「欲しい結果」を持たせる。
2. 人物は説明のためではなく、説得、拒否、隠蔽、確認、挑発、慰めのために話す。
3. 質問に正面から答えない、言い直す、黙る、動作で返すなどの人間的反応を使う。
4. 全員へ均等に一行ずつ割り当てない。
5. 口癖は一場面に必要な回数だけ使い、人格の代用品にしない。
6. 話者名を消しても、語彙、文の長さ、判断基準で判別できる状態を目指す。
7. 冗談は危機を無効化せず、恐怖に耐える人物の防衛反応として置く。
8. 作戦説明は短く、プレイ中に必要な情報だけにする。
9. 人物関係は呼び方、割り込み、視線、触れ方、沈黙で示す。
10. 一場面の終わりに、関係、情報、目的のいずれかを必ず変化させる。

### 4.4 構成

1. 序章：流出の夜と主人公の喪失。
2. 第一幕：生存者部隊と移動拠点の現在。
3. 第二幕前半：救助を通じて研究施設由来の証拠を得る。
4. 中間点：TAKUYAが単なる怪物ではなく、T計画の中心人物だったと判明。
5. 第二幕後半：西新駅地下へ進み、流出経路と研究輸送線を発見。
6. 終盤：人命救助と感染封鎖のどちらかではなく、両方を成立させる困難な作戦。
7. 章末：当面の危機を解決しつつ、大学病院とT計画の核心を次章へ残す。

### 4.5 ゲームとの統合

1. 会話で述べた危険は、その直後の敵、地形、目的として遊べるようにする。
2. 加入人物は会話だけでなく、戦闘中の行動で価値を見せてから加入する。
3. 勝利後に突然ユニット一覧へ追加しない。
4. 戦闘中の台詞は最大三往復を基本とし、操作を止めない。
5. 敗北・再戦台詞はヒントを含むが、同じ説明を繰り返さない。

### 4.6 サウンド

1. 導入は病院設備音、警報、遠い歓声、犬の吠え声、救急車、停電後の無音で崩壊を描く。
2. ウイルス流出を大音量の爆発だけで済ませず、換気音の停止、ガラス、ケージ、足音など因果の音を置く。
3. 西新の夜は店内音楽、呼び込み、瓶、雨、地下鉄の振動から、悲鳴、衝突、サイレンへ移行する。
4. 会話中はボイスまたは重要台詞を優先し、BGM・環境音を一時的に下げる。
5. 恐怖場面では無音を使用し、常時BGMで感情を命令しない。
6. 戦闘BGMは敵数ではなく、目的の段階、救助対象の危機、ボスの破損段階へ追従させる。
7. スマートフォン本体スピーカーでも、台詞、警告、攻撃音の優先順位が崩れないミックスにする。

## 5. 五つの仮想レビュー人格

### A. 日本語会話編集者
- 語尾、敬語、主語、省略、間が日本人の会話として自然か。
- 台詞が説明文を分割しただけになっていないか。
- 声に出した時に息が続くか。

### B. ゲーム・ナラティブデザイナー
- 会話後にプレイヤーが何をするか明確か。
- 台詞とステージ目的、敵、報酬、加入がつながっているか。
- 操作を止めすぎていないか。

### C. ホラー演出家
- 怖さの原因が説明過多で消えていないか。
- 平常、異変、発見、反撃の波があるか。
- 音、暗さ、見えない危険を使えているか。

### D. 群像劇脚本家
- 各人物が同じ情報を別の言い方で読むだけになっていないか。
- 人間関係が場面ごとに変化しているか。
- 加入・離脱・対立に感情的な理由があるか。

### E. サウンドディレクター
- BGM、環境音、SE、ボイスの開始、停止、ダッキングが指定されているか。
- 音が物語上の証拠または危険通知として機能しているか。
- スマートフォンで重要音が埋もれないか。

## 6. 不採用とする旧稿の問題

- 人の声を真似る感染体を物語全体の中心へ置いたため、ウイルス災害の王道的な恐怖と地域崩壊が見えなかった。
- 導入が短すぎ、主人公、街、惨事、移動拠点の関係が理解できなかった。
- 会話が状況報告と作戦確認に偏り、人物の欲望や関係がなかった。
- タイトル画面へ長いあらすじを置き、プレイ開始後の映像・音響で見せる責任を放棄していた。
- TAKUYA、大学病院、Tウイルス、西新駅が一つの因果でつながっていなかった。
- サウンドが台本の後付けで、場面設計に含まれていなかった。

## 7. 新稿の合格条件

- ゲーム開始後に導入シーンが始まり、スキップと再視聴が可能。
- 何が流出し、どこで惨事が起き、主人公がなぜ戦うかを初見で理解できる。
- Tウイルスの全真相は明かさないが、現時点の目的は迷わない。
- 各主要人物に、恐怖、欲望、弱点、他者との摩擦がある。
- 各会話を五人格で音読監査し、全員が承認するまで正本化しない。
- 台本内にBGM、環境音、SE、無音、ダッキングの意図を記録する。
