## 6. 画像の一枚単位承認制

すべての新規画像は、**一枚ごとにプロデューサーの明示承認を得る**。

対象：

- キャラクター基準アート
- ポートレート
- 会話立ち絵
- 表情差分
- 戦闘スプライトシート
- 敵スプライトシート
- ボス画像
- 背景
- ステージオブジェクト
- 重要イベント一枚絵
- その他、ゲーム画面へ表示する視覚アセット

運用：

1. Codexは同時に複数画像を承認依頼しない。
2. 一度に一枚だけ提示する。
3. 提示時に、asset ID、用途、予定パス、画像、短いデザイン意図、確認事項を添える。
4. 質問は`この画像で進めてよいですか`とする。
5. プロデューサーの回答は`承認`、`修正：内容`、`却下`のいずれかとして扱う。
6. 明示承認前に正式パスへ採用、ゲーム統合、派生画像制作、次の画像承認依頼へ進まない。
7. 修正版は同じasset IDの新revisionとして再提示する。
8. 却下画像を正式アセット、仮画像、背景素材へ流用しない。
9. 承認履歴をIssue #37またはDraft PR #38へ記録する。
10. すべての最終画像について、承認者、承認日時、revision、commit、最終パスをasset manifestへ記録する。

スムーズな進行：

- 画像承認待ちの間も、セーブ、AI、UI、戦闘ロジック、データ、テストなど画像非依存作業は並行継続する。
- 承認待ち画像に依存する統合工程だけを停止する。
- 画像承認は一枚ずつだが、毎回長い説明をせず、必要情報を固定フォーマットで短く提示する。
- 圧縮、ファイル形式変換、透明padding、pixel-identicalなリサイズなど、デザインを変えない機械的派生は新規デザイン承認とは扱わない。ただし最終ゲーム画面での表示QAは必須。
- 画像一覧をまとめて承認したものとして扱わない。

承認依頼テンプレート：

```md
## 画像承認依頼 1件

- Asset ID：
- 用途：
- 予定パス：
- Revision：
- デザイン意図：1〜3行
- 確認：人物同一性／世界観／役割識別／スマホ視認性／破綻の有無

[画像1枚]

この画像で進めてよいですか。
回答：`承認`／`修正：...`／`却下`
```

## 7. 台詞・人物像の修正権限

固定4名：

- パイセン
- クマバーソン
- ババヤガ
- クレイジーキング

この4名の人物像、口調、奇妙な言葉、代表的な言い回しを基本的に変更しない。

例外：

- 文脈につながらず意味が成立しない
- 同じ冗談を不自然に連打している
- 表示上の誤字、話者違い、設定矛盾
- ゲーム進行を誤認させる

例外修正でも、人物の特徴を一般的な自然文へ均してはいけない。修正理由と差分を記録する。

その他人物は、役割、基本属性、加入順、関係性、プロット、場面順を維持したうえで、不自然な台詞と性格描写を人間的に修正できる。実装後に話者別台詞一覧と音読監査結果を提出する。

## 8. 同一人物の複数召喚

同じキャラクターを戦場へ複数体同時に召喚できる現行ゲーム性を維持する。

- 編成カード自体は人物ごとに1種類
- 選択済みカードから、コストと再出撃待ちに従って同じ人物を繰り返し召喚可能
- 同時存在数を1体へ制限しない
- 戦闘不能後も再出撃可能
- 永久離脱、死亡持越し、人物ロックを通常モードへ導入しない
- 7枠編成内へ同じ人物カードを重複登録する必要はない

## 9. 前進・後退命令

0.7.0では`前進`・`後退`の追加操作を実装しない。関連UI、コマンドゲージ消費、ボタン、チュートリアルを追加しない。

代わりに味方AIを改善する。

必須ルール：

- 攻撃可能な敵が同一レーンまたはユニットの攻撃可能範囲にいる場合、敵を無視して前進しない
- 目の前の敵、接触中の敵、味方拠点を攻撃中の敵を優先して停止・迎撃する
- 敵とhitboxが重なる、または通過を阻まれている状態で前進状態を継続しない
- TAKUYA撃破後も、残存敵、後続敵、CRAWLERを攻撃する敵を有効標的として扱う
- Y座標ずれ、レーン移動、target消失後に索敵を失わず、次のtickで再取得する
- 遠距離役、近接役、回復役ごとの攻撃可能条件と索敵条件を一致させる
- 味方が攻撃せず敵を素通りして死亡し、敵が増え続ける状態を再発させない

必須回帰テスト：

- Stage 1〜3
- 全既存ユニット種
- 3レーン
- 敵が正面、背後、CRAWLER付近、レーン移動中
- Stage 3 TAKUYA撃破直後
- 残存敵と後続敵が同時にいる状態
- 味方が前進中に敵が射程へ入る状態
- 近接、遠距離、医療支援の別

実ブラウザQAで、味方が敵を無視して前進する再現が0件であることを確認する。

## 11. アート以外の実装進行

画像承認は一枚単位で停止するが、0.7.0全体は丸投げで実装まで進める。

- Codexは画像以外の技術判断を自律解決する
- 実装漏れ、文書齟齬、参照切れを自分で横断監査する
- スマートフォンを最優先とする
- PCは基本機能の回帰確認に留める
- 844×390、844×340、物理iPhone Safariをリリース阻害基準とする
- safe area、ブラウザUI、回転、画面ロック、タブ復帰、タップ領域を確認する
- 有料で提供できる水準を目標とし、仮画像、未完成UI、豆粒表示、読めない文字、誤タップ、レーン外歩行、音声残留、セーブ消失を残さない
