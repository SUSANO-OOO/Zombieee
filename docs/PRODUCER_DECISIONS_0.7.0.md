- Codexがデザイン、生成、加工、スプライト化、実装まで担当
- 日本の商業2Dゲーム、青年漫画、ダークアクション漫画を基礎とする
- フォトリアル、実写風、プラスチック質感の3D、幼い萌え、AI量産顔は禁止
- 顔、髪、体格、配色、武器、姿勢、シルエットを人物ごとに変える
- スマートフォン縮小画面で判別できること
- プレイヤーが使いたいと思える魅力を持たせる
- 敵は人間由来で、西新・地下鉄・病院の世界観と能力が外見から分かること
- 戦闘スプライトの制作基準方向は右向き

## 7. 画像承認

すべての新規画像は一枚ごとの明示承認制。

1. 一度に一枚だけ提示
2. Asset ID、用途、予定パス、revision、短い意図を記載
3. `この画像で進めてよいですか`と確認
4. `承認`前に正式採用、派生制作、統合、次画像提示へ進まない
5. 修正は同Asset IDの新revision
6. 却下画像は再利用しない
7. 承認履歴をIssue #37またはPR #38とasset manifestへ記録

承認待ち中もsave、AI、UI、戦闘ロジック、テストなど画像非依存作業は並行継続する。

## 10. 手動戦術と味方AI

前進、後退、防御、標準、突撃など、同目的の手動戦術機能はUI、入力、説明、状態、保存項目ごと撤去する。

ユニット配置コスト、再出撃待ち、支援ゲージなど別用途の仕組みは残す。

味方AI：

- 攻撃可能な敵を無視して前進しない
- 接触中、通行阻害、CRAWLER攻撃中の敵を優先
- hitbox重複中に前進しない
- TAKUYA撃破後も残存・後続敵を再索敵
- Yずれ、レーン移動、target消失後に次tickで再取得
- 実ブラウザで敵素通り再現0件

敵素通り、非現実的な接触、表示とhitboxの不一致は公開阻害条件。
