## 3. プレイヤー主人公の固定条件

- 主人公はプレイヤー。
- 発生から四十三日後、プレイヤーはCRAWLERの指揮官である。
- プレイヤーは基本的に無言。固定台詞、性格選択、会話分岐を実装しない。
- プレイヤー名入力は脚本の必須条件にしない。UI上の既定表示は`指揮官`で成立する。
- 台詞へプレイヤー名変数を挿入しない。名前入力機能を後から追加しても、脚本データを変更せず成立させる。
- 指揮官の存在は、報告先、視線、作戦卓、既存の出撃・帰還・マーカー操作で示す。

## 4. ゲーム構造の固定条件

- プロローグは崩壊前の小料理屋`くまや`から始める。
- 研究所・研究員の会話シーンは表示しない。
- プロローグに戦闘、操作チュートリアル、追加ステージを作らない。
- 最初の戦闘は既存の`Stage 1 西新商店街`。
- Stage 1〜6の既存目的、敵構成、勝利条件を脚本都合で変更しない。
- 会話選択肢による本筋分岐、好感度、フラグ、別エンディングを追加しない。
- 物語イベントの登場人物は7枠編成と分離してよい。戦闘中barkは実際に出撃している人物だけ再生する。
- 作戦説明と結果数値はUIへ分離し、人物へゲーム仕様を朗読させない。

## 5. 会話品質の固定条件

- 人物はプレイヤーへ報告し、指示を待ち、結果を返す。プレイヤーを傍観者にしない。
- 台詞は一画面で読める長さを基本とする。
- 冗談は場面を止めず、人物の価値観から出す。
- パイセンの語録は決め場面に限定し、連打しない。
- クマバーソンは熱血・パワー・食事を軸にする。
- ババヤガは基本冷静。崩れると博多弁が出る。意味不明な比喩は一場面に一つ以下。
- クレイジーキングは漢字とカタカナ中心、二言三言で話す。長い説明、通常口調、連続した冗談を与えない。
- ノイズは必要な情報だけを指揮官へ報告し、製品仕様の説明係にしない。

## 6. 実装規則

- fresh saveで`物語を始める`を押した直後、プロローグを再生する。
- プロローグは初回自動再生、スキップ、回想からの再視聴に対応する。
- スキップ時は短い固定要約からCHAPTER 1へ進む。
- Codexは台詞、場面順、人物関係を要約、言い換え、追加、削除しない。
- BGM、環境音、SE、無音、ダッキングを台本の一部として実装する。
- 正式採用は、実装後の音読、実音源、実ブラウザ、物理スマートフォン受入後とする。
