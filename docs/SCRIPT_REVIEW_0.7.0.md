## 3. 五視点レビュー

### 3.1 製品設計

判定：条件付き合格。

- 0.7.0の中心体験はユニット収集と7枠編成で一貫している。
- 11名、3プリセット、キャップ調達、新敵4種、地下3ステージは同一更新として成立する。
- レベル、装備、ガチャ、4〜5レーンを非対象にしたため、範囲膨張を抑えている。
- 数値はCodexへ委任したが、比較基準、試験、記録条件があるため無制限な独断にはなっていない。

### 3.2 ナラティブ

判定：条件付き合格。

- プレイヤーが大衆居酒屋「くまや」の常連客として導入される。
- 発生夜の避難とCRAWLER確保を通じ、四十三日後の指揮官へ接続できる。
- プロローグに追加戦闘を置かず、最初の戦闘をStage 1に維持する。
- TAKUYA、T計画、西新駅地下への因果を維持する。
- 各PART台本はv7旧表記を含むため、実装前のv8正規化が必要。

### 3.3 キャラクター・アート

判定：実装条件付き合格。

- パイセン、クマバーソン、ババヤガ、クレイジーキングは保全対象。
- その他7名とノイズは全面再設計対象。
- レイダーへの改名が固定された。
- 医療支援ユニットの正式名はCodexが決定し、理由を記録する。
- 顔、髪、体格、色、武器、姿勢の差別化が受入条件になっている。
- フォトリアル、実写、3Dプラスチック肌、同一AI顔、別人流用は禁止。
- 実画像ができるまで最終的な魅力度判定はできない。

### 3.4 技術・セーブ

判定：条件付き合格。

- 現在の0.7.0 branchは`main@f4cf603e8e003c8fa14b2b7368700ed7a6501261`より後方にあり、PR #39の取込みが必須。
- PR #39はlocalStorageの永続化確認、IndexedDB二重保存、消失時復元、保存不能警告を追加した。
- 現方式はlocalStorageに破損文字列が残る場合のIndexedDB優先復旧が不足するため、両保存候補の独立検証を追加する。
- schema v5、revision、updatedAt、完全性確認、旧save退避、migration失敗時の非上書きを必須とする。

### 3.5 スマートフォン・戦闘画面

判定：条件付き合格。

- 844×390、844×340、物理iPhone Safariを必須受入に含めた。
- 7カードを単純縮小して豆粒化することを禁止した。
- 背景制作前のレーン固定、床面との一致、anchor・hitbox debug表示を要求した。
- 同じ背景の色替えだけでStage 4〜6を済ませることを禁止した。
- 実背景と実スプライトがないため、現時点では最終合格にできない。
