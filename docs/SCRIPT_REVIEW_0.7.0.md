# 西新世紀末物語 — 0.7.0脚本レビュー

更新日：2026-07-17  
対象：`outbreak-origin-v7`

## 1. 判定

**実装候補としてGitHubへ掲載可。正式採用は未完了。**

掲載可とした理由：

- 主人公がプレイヤーであることが脚本と実装条件で一致した。
- プレイヤーは無言で、性格選択、会話分岐、固定台詞を要求しない。
- プロローグに独自戦闘や操作チュートリアルを追加していない。
- 最初の戦闘は既存のStage 1。
- Stage 1〜6の既存構造を脚本都合で変更していない。
- Codexがデータへ落とし込めるevent ID、固定順序、条件付きbark、UI分離が明記されている。

正式採用に必要なもの：

- 実装後の音読監査
- 実ブラウザでの表示テンポ確認
- 実音源を使った聴感確認
- 844×390、844×340、PC 1280×720、物理iPhoneでの受入

## 2. 仮想人格レビュー

以下は実在人物ではなく、異なる利用者・制作者の視点を分離したレビュー観点である。

### 2.1 脚本家

確認項目：

- 場面ごとに誰が何を求めているか
- 台詞が説明ではなく反応として出ているか
- ユーモアが物語を止めていないか

判定：合格。

修正内容：

- 研究所会話を全面削除。
- `くまや`の日常から感染発生へ直接移行。
- 長い説明台詞を削り、原因情報は緊急速報、搬送箱、端末記録へ分散。
- 感傷的な独白を減らし、人物の短い行動と会話へ置換。

### 2.2 初見プレイヤー

確認項目：

- 自分が物語の中心だと分かるか
- プロローグからStage 1までが長すぎないか
- 選択肢を読まないと進めない構造になっていないか

判定：合格。

修正内容：

- 四十三日後は全報告を`指揮官`であるプレイヤーへ向ける。
- プレイヤーは既存の作戦卓、出撃決定、帰還操作で判断を示す。
- 会話選択肢と性格分岐を全廃。
- プロローグに追加戦闘を置かず、Stage 1を最初の戦闘として維持。

### 2.3 ゲーム・ナラティブデザイナー

確認項目：

- 物語と既存ゲームループが衝突していないか
- 編成キャラと物語登場キャラの矛盾が起きないか
- ステージ目的が会話で改変されていないか

判定：合格。

修正内容：

- 物語イベント登場人物と7枠編成を分離。
- 戦闘中barkは実際に出撃している人物だけ再生。
- 作戦説明と報酬数値はUIへ分離。
- Stage 1〜6の目的、敵構成、勝利条件を変更しない。

### 2.4 Codex実装担当

確認項目：

- 分岐フラグが不要か
- 変数挿入や複雑な条件が不要か
- event ID、再生条件、停止条件が明確か

判定：合格。

実装条件：

- 固定順序のeventデータとして実装可能。
- プレイヤー名変数を台詞へ挿入しない。
- 条件分岐は`該当キャラが出撃中か`だけを基本とする。
- 敗北、再戦、スキップ、回想では音源とイベント状態を初期化する。
- 台詞の要約、言い換え、追加は行わない。

### 2.5 キャラクター会話編集

確認項目：

- 話者名を隠しても区別できるか
- キャラの特徴が口癖の連打になっていないか
- 寒いツッコミ合戦になっていないか

判定：条件付き合格。

固定条件：

- パイセンの語録は出撃・危機など決め場面に限定。
- クマバーソンは熱血、パワー、食事を軸にする。
- ババヤガは基本冷静。崩れた時だけ博多弁。奇妙な比喩は一場面に一つ以下。
- クレイジーキングは漢字とカタカナ中心、二言三言まで。長台詞禁止。
- 新規5名、ガンテツ、モンキー、ノイズは実装後の音読で最終調整する。

## 3. プレイヤー名入力の判断

0.7.0脚本では必須にしない。

理由：

- 入力欄、保存、禁止文字、未入力時表示、migrationを追加すると実装範囲が増える。
- 台詞内で名前を呼ばないため、現時点の体験向上が小さい。
- UI表示`指揮官`で物語とゲームプレイが成立する。

後から名前入力を追加する場合も、作戦画面やセーブ表示だけで使用し、台詞データへの変数挿入を要求しない。

## 4. 残存リスク

- `センセイ`を含む新規人物名・人物像は、正式アート制作前に別途プロデューサー確認が必要。
- 文章上のテンポが良くても、スマートフォンの1画面表示量で遅くなる可能性がある。
- ユーモアは音声、間、表情がない状態では硬く見える可能性がある。
- 正式音源とキャラクター画像が揃うまで、没入感の最終評価はできない。

## 5. 結論

`outbreak-origin-v7`は、Codexへ渡す実装候補として掲載する。

ただし、文書がGitHubに存在するだけでは完成扱いにしない。実装後の音読、表示、音響、物理スマートフォン受入で不合格が出た場合は、同じfeature branch上で修正する。
