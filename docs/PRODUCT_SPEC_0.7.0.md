## 6. アート担当

キャラクターおよび敵の正式アートはChatGPT／プロデューサー側で生成・選定・承認する。

Codexの担当は次に限定する。

- アセットmanifest
- パスと内部IDの紐付け
- crop、透過、描画順、スプライト分割
- 肖像と戦闘表示の統合
- 当たり判定、アニメーション、性能監査

未承認の自作画像、別人物の顔・身体・武器を流用した画像、単色人形、丸・四角、低品質な仮素材を正式成果にしない。承認済み画像が未提供の対象は不足assetとして明記し、Draft PRを完成扱いにしない。パイセンの元素材は非破壊で保全する。

## 7. 所有・調達

- 初期所有4名
- ステージ進行、物語、星合計などで調達候補を追加
- 物語加入で無料所有する人物と、調達権だけ解放する人物を分ける
- 新規ガンテツは物語加入
- 新規モンキーは地下区域進行後に調達可能
- ガチャ、時間更新、重複購入、売却、返金なし
- 購入確認、条件不足、キャップ不足、二重購入防止
- 既存解放人物は対応する新人物の所有済みへmigration
- 既存キャップ、星、進行、既読、音響設定を保持

## 8. 編成・プリセット

- 所有ユニットから最大7名
- 1名以上で出撃可能
- 同一人物の重複登録禁止
- 同一クラスの別人物は複数編成可能
- 編成順を戦闘HUDのカード順へ反映
- PCはドラッグ操作、スマートフォンは左右移動操作を併設
- プリセット3個、名称変更可能
- 人物、順序、戦場物資を保存
- ステージごとに直前に使用したプリセットを勝敗に関係なく記憶
- migrationや所有状態変更で無効になった人物を安全に除外

## 9. 西新駅地下3ステージ

1. `西新駅・改札区域`：感染中継点破壊と生存者経路確保
2. `西新駅・ホーム／線路区域`：保守台車の護衛と物資回収
3. `西新駅・保守トンネル／封鎖区域`：防衛、封鎖成立、退路確保、脱出

- 改札、駅務室、ホーム、線路、車両、保守設備、トンネル、封鎖扉を画面だけで識別可能にする
- 3ステージを同一背景の色替えで済ませない
- 背景、歩行面、3レーン、地形、当たり判定、draw orderを一致
- 正式な新ボスは追加しない
- 改札喰いは大型特殊個体として専用HP・破損段階を持つ
- 航空支援は地下でも通常利用可能
- 第二支援、地下専用支援は追加しない

## 10. サウンド

BGM、環境音、物語SE、UI SE、戦闘SE、短いキャラクターボイス、無音、ボイスダッキングを一体で設計する。

- 台本指定の開始・停止・復帰を実装
- 通常会話中はBGMを3〜5dB下げる
- 重要台詞では環境音を1〜3dB下げる
- 台詞終了後250〜500msで復帰
- スキップ後に前場面のループを残さない
- 重要警告、撤退、救助対象危機を攻撃SEより優先
- 同時発音数、クリップ、AudioContext初回解除・復帰・二重再生を監査
- スマートフォン本体スピーカー、イヤホン、PCで確認
- 仮音源や低品質音源を正式完成扱いにしない

## 15. 停止境界

実装ミッションは同じfeature branchとDraft PR上で、調査、実装、テスト、通常commit、通常push、CI、隔離プレビュー、独立レビューまで進められる。

以下は別のリリース承認まで禁止する。

- Ready化
- merge
- `main`変更
- tag
- GitHub Release
- ChatGPT Sites正式公開
- Issue close
- branch削除

文書、コード、仮画面が存在するだけで完成扱いにしない。正式アート、実音源、実ブラウザ、物理スマートフォンの受入が不足する場合は、未完成として停止する。
