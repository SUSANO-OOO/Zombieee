## 3. 対象

### 3.1 ユニット所有と調達

ユニット状態を次の4段階で管理する。

1. 未遭遇
2. 情報判明
3. 調達可能
4. 所有済み

- ガチャ、ランダム抽選、時間更新ショップは使用しない。
- 調達品はステージ進行、物語、星条件によって追加する。
- 同一ユニットを複数回購入できない。
- 売却・返金は実装しない。
- 通常ステージの再プレイだけで全ユニットを取得可能にする。
- 初期・物語加入・直接解放・キャップ購入を人物ごとに使い分ける。
- 購入前に人物、主要クラス、役割、固有特性、価格、解放条件を確認できる。

### 3.2 通貨

現行のプレイヤー向け名称`補給物資`を、0.7.0では**キャップ**へ変更する。

- UI例：`所持キャップ 320`、`必要キャップ 450`、`120キャップ獲得`
- 既存saveの`supplies`はschema migrationで`caps`へ1対1で保持する。
- 内部互換のため旧フィールドを読み取れるようにし、旧saveを初期化しない。
- キャップは西新の飲食店・避難拠点で生まれた交換証票として世界設定へ組み込む。
- 通常の瓶蓋と区別するため、回収・検品後に簡易刻印されたキャップを流通単位とする。
- 0.7.0では第二の恒久通貨を追加しない。

### 3.3 プレイアブル規模

0.7.0の正式プレイアブルは**11名**とする。

- 0.6.5の9種類の戦闘役割は維持する。
- 新規味方として重装防御役1名、工兵1名を追加する。
- パイセン、クレイジーキング、クマバーソン、ババヤガは人物・外見・武器の正本を維持する。
- 橘迅、黒木凛、白石直人、大庭豪、真壁玲奈は人物、名称、肖像、戦闘アートを全面差し替えする。
- 差し替え後も遊撃、射撃、衛生、破砕、制圧という戦術上の役割は継承する。
- 水城奈々は廃止し、非戦闘の専任オペレーターへ全面差し替えする。
- 個別設定は[0.7.0キャラクター正本](CHARACTERS_0.7.0.md)を使用する。

### 3.4 データ構造

人物、戦闘ロジック、クラス、表示名を分離する。

- `unitId`：人物を識別する永続ID
- `combatKind`：既存戦闘ロジックとの接続点
- `classId`：主要クラス
- `roleTags`：複数の役割タグ
- `displayName`：プレイヤー向け人物名
- `classDisplayName`：プレイヤー向けクラス名

表示名やクラス表示を変更しても、永続IDとセーブ互換を壊さない。現在の`brawler`、`scout`等を人物IDとして使い続けず、schema v5 migrationで新しい`unitId`へ明示的に対応付ける。

### 3.5 主要クラスと役割タグ

主要クラスは7種類前後とし、各人物は主要クラスを1つだけ持つ。

候補基準：

- 前衛
- 遊撃
- 射撃
- 重装
- 支援
- 工兵
- 狂戦

同じクラスでも、標的優先、攻撃方法、射程、押し返し、範囲、固有特性を変える。役割タグは例として`近接`、`高耐久`、`対大型`、`対群体`、`回復`、`足止め`、`高機動`、`制圧`を使用できる。

### 3.6 固有特性

全11名に、編成理由になる固定の固有特性を1つ持たせる。

- 固有特性は自動発動または常時効果とする。
- プレイヤーが任意発動する個別アクティブスキルは0.8.0へ送る。
- 専用ゲージや人物ごとの別操作体系を0.7.0では導入しない。
- 数値差だけで終わらせず、実戦で使い方が少し変わる強さにする。

### 3.7 能力表示

一覧カードでは判断しやすい段階表示を使う。

- 耐久
- 火力
- 機動
- 支援

詳細画面では次の具体値を表示する。

- HP
- 攻撃力または回復量
- 攻撃間隔
- 射程
- 移動速度
- 出撃コスト
- 再出撃時間
- 押し返し、制圧、特殊対象補正など人物固有の重要値

レベル、成長値、装備状態は0.7.0では表示しない。将来追加できるレイアウト余地だけを残す。

## 5. 7枠編成とプリセット

### 5.1 編成

- 所有済み11名から最大7名を選択する。
- 1名以上なら空き枠を残して出撃できる。
- 同一人物を複数枠へ登録できない。
- 同じ主要クラスの別人物は複数登録できる。
- 編成順を戦闘HUDのカード順へ反映する。
- 戦闘中は同じ人物カードから、指揮力と再出撃時間が許す限り反復召喚できる。
- 戦場上の同一人物複数存在は通常モードの抽象表現として維持する。

### 5.2 操作

- PCではドラッグによる並び替えを許可する。
- スマートフォンではドラッグだけに依存せず、左右移動ボタンまたは同等に明確な操作を併設する。
- タップ領域、フォーカス、キーボード操作、スクリーンリーダー向け名称を整備する。

### 5.3 プリセット

プリセットは3個とする。

保存対象：

- 編成人物
- 編成順
- 選択中の戦場物資
- プリセット名

航空支援とCRAWLER一斉掃射は固定能力であり、選択肢がないためプリセットへ冗長保存しない。削除する作戦方針も保存しない。

- プレイヤーはプリセット名を変更できる。
- ステージごとに直前に出撃したプリセットを記憶する。
- 勝敗に関係なく、直前に実際に使用したプリセットを次回初期選択する。
- 所有状態変更やmigrationで無効な人物があれば、安全に除外して通知する。

## 6. 前進・後退指示

現行の`防衛・均衡・攻勢`は削除する。名称と実挙動の違いが分かりにくく、プリセットに保存する価値も弱いためである。

代わりに、戦闘中の短時間命令として次を実装する。

- `前進`
- `後退`

通常時は既存AIが自動判断する。

- 前進：一定時間、担当レーンの前線押し上げと敵拠点接近を優先する。
- 後退：一定時間、CRAWLER側へ下がり、危機対象の迎撃と隊列再構築を優先する。
- 命令は永続トグルにせず、短い有効時間と再使用待ちを持つ。
- CRAWLERを攻撃中の敵を無視する、後退したまま戻らない、頻繁に方向転換する回帰を禁止する。
- UI表記は短い日本語と方向アイコンに限定する。

## 7. 戦場物資・航空支援・一斉掃射

- 戦場物資は投下ポッド、爆薬ドラム、救護所から1種類を選ぶ構造を維持する。
- 0.7.0では戦場物資の購入・強化を追加しない。
- 航空支援は現在どおり固定能力として維持し、第二支援を追加しない。
- 西新駅地下でも航空支援は使用可能とする。
- 地下専用支援を追加しない。
- CRAWLER一斉掃射は航空支援とは別の固定能力として維持する。
- プレイヤー画面へ不要な上位カテゴリ名を押し付けず、`航空支援`、`一斉掃射`と具体名で表示する。

## 8. 西新駅地下鉄区域

### 8.1 全体

西新防衛線クリア後、エリアマップ上に西新駅の地下鉄区域3ステージを追加する。

1. 西新駅・改札区域
2. 西新駅・ホーム／線路区域
3. 西新駅・保守トンネル／封鎖区域

物語の入口は、繰り返し救難信号の下に混ざる生存者の断続的な実音声と、地下から地上へ流出する感染反応の調査である。

### 8.2 ステージ1：改札区域

- 目的：感染中継点を破壊し、閉じ込められた生存者への経路を開く。
- ガンテツを物語上で発見する。
- 改札、シャッター、券売機、案内板、階段、地上光を画面上で識別できる。
- 既存の西新商店街背景を暗くしただけの画像は禁止する。

### 8.3 ステージ2：ホーム／線路区域

- 目的：生存者と回収物資を載せた保守台車を護衛する。
- 物資回収ミッションを兼ねる。
- ホーム、線路、停車車両、電光表示、転落防止柵、トンネル口を識別できる。
- 台車、護衛対象、敵の出現点、勝敗判定を一致させる。
- モンキーの存在と調達条件を物語へ接続する。

### 8.4 ステージ3：保守トンネル／封鎖区域

- 目的：感染源を封鎖する準備が整うまで防衛し、退路を確保して脱出する。
- 正式ボスは追加しない。
- 大型特殊個体`改札喰い`を中核脅威として配置する。
- トンネル、保守通路、電源設備、封鎖扉、配管、退路を識別できる。
- 防衛完了だけで即勝利せず、脱出または封鎖成立までの因果関係を画面と会話で示す。

### 8.5 背景と描画

3ステージは色替えや同一構図の小変更で済ませない。

- カメラ構図
- 歩行面
- レーン中心
- 前景・後景
- 光源
- ランドマーク
- 敵出現口
- 破壊対象
- 勝利演出

を別設計にする。キャラクターが壁、空間、線路外、遠景を歩いて見えないよう、背景、地面、当たり判定、draw orderを一致させる。

## 9. 新規敵

新規敵3種類は完全に別のスプライト、輪郭、移動、攻撃、死亡表現を持つ。

### 9.1 絡手

- 捕縛型
- 長い腕、つり革、ケーブル状の感染組織で味方1体を一時拘束する。
- 拘束中も対象を即死させず、仲間からの攻撃または時間経過で解除できる。
- 連続拘束、操作不能の長時間固定、画面外拘束を禁止する。

### 9.2 漏泥

- 地形妨害型
- 地面へ感染性の液体または腐食域を残す。
- 同じ場所へ密集する戦術を崩すが、回避不能な全画面床ダメージにしない。
- 予告、実範囲、持続、消滅を視覚的に一致させる。

### 9.3 走鬼

- 低耐久・極端な高速型
- 既存runnerよりさらに突破能力へ特化する。
- 単なる速度違いで終わらせず、前線1体を飛び越える、線路側から侵入する等の固有挙動を一つ持つ。
- 命中率乱数だけで対処しにくくせず、早い発見、遊撃、後退指示、配置で対処できる。

### 9.4 改札喰い

- 正式ボスではない大型特殊個体
- 改札機、柵、案内板などを感染組織へ取り込んだ外見
- 高耐久、押し返し、前面装甲の破損段階を持つ
- HP、損傷、当たり判定、弾着、死亡演出を一致させる
- TAKUYA級の専用章ボス演出、長大な多段階戦、正式図鑑ボス扱いは行わない

## 10. エリアマップ

現行マップを、CRAWLER内部の大型作戦卓として全面改善する。

- 現実の西新周辺地図をゲーム向けに再構成する。
- 平面的なノードだけでなく、道路、街区、ランドマーク、立体的な市街要素を組み合わせる。
- 西新駅、地下鉄路線、既存3ステージ、CRAWLER現在地の関係を理解できる。
- 西新駅の地下3ステージは地上マップ上へ直接並べる。
- 複雑な別マップ遷移や長い展開アニメーションを必須操作にしない。
- 選択中の地下区間だけが明確に浮かぶ軽い演出は許可する。
- 844×390および844×340で、ノード名、星、封鎖、選択状態、編成への導線が読める。

## 11. キャラクターアートと音声

- 新規2名、差し替え5名、新オペレーター、新敵3種、大型特殊個体は固有アートを必須とする。
- 他キャラクターの顔、身体、武器、輪郭が混入したように見える流用を禁止する。
- 肖像と戦闘スプライトで、髪、顔、体格、服、武器、色、主要装備を一致させる。
- 暫定の丸、四角、シルエット、単色人形を本番表示として残さない。
- パイセンの保全条件を維持する。
- 全11名に出撃、攻撃、被弾、撤退または戦闘不能の短い人物別音声を持たせる。
- 音声同時再生数、ダッキング、スマホスピーカー、イヤホンでの突出を確認する。

## 12. ストーリー

- 正式台本は[0.7.0完全台本](SCENARIO_0.7.0_COMPLETE.md)を唯一の正本とする。
- Codexは台本、人物性格、会話を独自生成・要約・改変しない。
- 既存5名とオペレーターの差し替えに合わせ、序章から西新防衛線までの既存会話も人物名だけの機械的置換にしない。
- 会話は説明の読み上げだけにせず、人物ごとの判断、恐怖、苛立ち、冗談、利害、関係性を持たせる。
- ブラックユーモアは緊張を壊さない位置に限定する。
- 戦闘中の通信は短く、操作を長時間止めない。
- 既読スキップ、会話ログ、戦闘イベント表示設定を維持する。

## 14. 手動バックアップ

クラウドセーブとは別に、プレイヤー自身がセーブを保存・復元できる機能を追加する。

配置：

- タイトル画面
- 設定画面

方式：

- JSONファイルの書き出し・読み込み
- 文字列のコピー・貼り付け

バックアップにはschema version、ゲームバージョン、作成日時、整合確認用情報を含める。

- 読み込み前に内容を検証する。
- 無効ファイルで現在のセーブを壊さない。
- 読み込み時は上書き内容を確認表示する。
- 読み込み成功後に再読込して同じ状態を復元できる。
- 秘密情報や端末識別情報を含めない。

削除対象：

- 星
- キャップ
- 所有人物

## 16. 難易度・経済
