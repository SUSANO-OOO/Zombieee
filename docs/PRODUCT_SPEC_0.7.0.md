## 4. プレイヤー主人公

- 主人公はプレイヤー。
- 発生から四十三日後はCRAWLERの指揮官。
- 基本的に無言。固定台詞、性格選択、会話分岐なし。
- プレイヤー名入力は必須にしない。UI既定表示`指揮官`で成立する。
- 名前入力を後から追加する場合も、台詞へ変数名を挿入しない。
- 報告先、視線、作戦卓、既存の出撃・帰還操作で物語の中心性を示す。

## 5. 脚本とゲーム構造

- プロローグは崩壊前の小料理屋`くまや`から開始。
- プロローグに戦闘、操作チュートリアル、追加ミッションを作らない。
- 最初の戦闘は既存のStage 1。
- Stage 1〜6の目的、敵構成、勝利条件を脚本都合で変更しない。
- 会話選択肢による分岐、好感度、別エンディングなし。
- 物語イベントの登場人物と7枠編成は分離可能。
- 戦闘中barkは実際に出撃中の人物だけ再生。
- UIで示せる作戦情報と結果数値を人物へ朗読させない。
